# 產品需求文件 (PRD) - 食譜收藏夾

## 1. 專案概述

### 背景與動機
許多喜歡下廚的人經常在網路上尋找食譜，或是自行研發拿手菜，但缺乏一個集中且系統化管理的工具。紙本筆記容易遺失、手機截圖難以搜尋。這個專案旨在打造一個「食譜收藏夾」，讓使用者能輕鬆數位化收藏與管理私人食譜，建立專屬的雲端廚房秘笈。

### 目標用戶
喜愛下廚、希望有條理地整理自己拿手菜菜單、或是習慣記錄烹飪心得的個人用戶。

### 核心價值主張
提供一個簡單、直覺的食譜管理平台，讓記錄材料與步驟變得輕鬆無負擔，並且隨時隨地都能快速查閱，提升烹飪體驗。

---

## 2. 功能需求

### 1. 新增食譜
- 作為一名喜愛下廚的用戶，我希望能夠新增食譜的菜名與簡介，以便我能快速建立一道新菜色的基本紀錄。

### 2. 材料清單管理
- 作為一名喜愛下廚的用戶，我希望能夠在每道食譜中列出所需的各項材料與精確份量，以便我在採買食材時不會遺漏任何項目。

### 3. 製作步驟管理
- 作為一名喜愛下廚的用戶，我希望能夠分點條列詳細的烹飪流程，以便我在實作時能一步一步跟著操作。

### 4. 顯示所有食譜列表
- 作為一名喜愛下廚的用戶，我希望能夠在一個畫面上看到所有已收藏的食譜菜名列表，以便我不知道該煮什麼時能快速瀏覽和選擇。

### 5. 查看食譜詳情
- 作為一名喜愛下廚的用戶，我希望點擊食譜列表中的某道菜後，能看到該菜色的完整資訊（包含簡介、材料清單與製作步驟），以便我開始準備烹飪。

---

## 3. 非功能需求

### 技術限制
- 後端框架：Flask
- 前端模板：Jinja2 (搭配 HTML/CSS/JS)
- 資料庫系統：SQLite

### 效能與安全考量
- **效能**：考量到個人使用的情境，查詢與顯示食譜的速度應盡量快速；SQLite 足以應付此規模的資料讀寫。
- **資訊安全**：
  - 防止 SQL Injection 攻擊，資料庫操作需使用 Parameterized Queries 或採用 ORM（如 SQLAlchemy）。
  - 對使用者輸入的文字進行跳脫處理，防止 XSS (Cross-Site Scripting) 攻擊，確保渲染食譜內容時的安全性。

---

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have（必須具備）
- 首頁列表顯示所有已收藏的食譜菜名。
- 新增食譜頁面，可填寫菜名、簡介、多項材料與份量、多個製作步驟。
- 單一食譜詳情頁面，可完整展示食譜內容。

### Should Have（應該具備）
- 修改/編輯已建立的食譜內容。
- 刪除食譜功能。
- （UI/UX）響應式網頁設計（RWD），讓使用者在廚房用手機看食譜時畫面不會跑版。

### Nice to Have（有了更好）
- 食譜圖片上傳功能（為每道菜加上封面圖）。
- 搜尋與分類標籤（例如：台式、甜點、家常菜），方便快速找菜。
- 一鍵產生採買清單的功能。
